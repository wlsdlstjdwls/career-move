# 코딩테스트 실전 연습 리스트

`study-plan.md` 섹션 1("코딩테스트")의 실행판. 6년차 기준 목표는 "다 풀기"가 아니라 "정해진 시간 안에 정확한 풀이를 고르고, 왜 그 자료구조/알고리즘을 골랐는지 말로 설명하는 것" — 아래 문제는 유형별 대표 예시이며, 실제 풀 때는 플랫폼에서 최신 난이도 태그를 다시 확인할 것(문제 난이도는 시간이 지나며 재조정되기도 함).

## 유형별 연습 순서 (우선순위 3단계, 프로그래머스 기준)

CJ올리브네트웍스는 프로그래머스 플랫폼·Lv1~2 위주·마지막 문제만 Lv3 정도로 살짝 올라가는 구성(37번 줄 리서치 참고). 우선순위1을 완벽히 끝내는 게 합격 확률에 제일 효과적 — 우선순위3은 시간 남을 때만.

### 우선순위 1 — 반드시 (해시/스택큐/구현/정렬/이분탐색)
- [ ] 완주하지 못한 선수 (해시)
- [ ] 의상 (해시)
- [ ] 베스트앨범 (해시+정렬)
- [ ] 올바른 괄호 (스택/큐)
- [ ] 기능개발 (스택/큐)
- [ ] 프로세스 (스택/큐)
- [ ] 같은 숫자는 싫어 (구현)
- [ ] 문자열 압축 (구현)
- [ ] 신고 결과 받기 (구현)
- [ ] 개인정보 수집 유효기간 (구현)
- [ ] 달리기 경주 (구현)
- [ ] H-index (정렬)
- [ ] 가장 큰 수 (정렬)
- [ ] K번째 수 (정렬)
- [ ] 입국심사 (이분탐색)
- 연습 포인트: HashMap을 언제 O(n)으로 쓰는지, 배열 대신 스택/큐를 왜 골랐는지(앞쪽 삭제 O(n) vs O(1)), 이분탐색을 쓰는 신호(정렬된 데이터 + 최적값 탐색)를 말로 설명

### 우선순위 2 — 가장 많이 나오는 유형 (BFS/DFS·그리디·힙)
- [ ] 타겟 넘버 (BFS/DFS)
- [ ] 게임 맵 최단거리 (BFS/DFS)
- [ ] 네트워크 (BFS/DFS)
- [ ] 단어 변환 (BFS/DFS)
- [ ] 여행경로 (BFS/DFS)
- [ ] 체육복 (그리디)
- [ ] 조이스틱 (그리디)
- [ ] 구명보트 (그리디)
- [ ] 더 맵게 (힙)
- [ ] 디스크 컨트롤러 (힙)
- [ ] 이중우선순위큐 (힙)
- 연습 포인트: BFS는 "최단 거리류", DFS는 "모든 경로 탐색/조합류" 선택 기준부터 말하기. 힙은 "매번 최댓값/최솟값을 O(log n)에 꺼내야 할 때" 신호

### 우선순위 3 — 시간 남으면 (DP·그래프·완전탐색)
- [ ] N으로 표현 (DP)
- [ ] 정수 삼각형 (DP)
- [ ] 가장 먼 노드 (그래프)
- [ ] 순위 (그래프)
- [ ] 피로도 (완전탐색)
- [ ] 모음사전 (완전탐색)
- 연습 포인트: DP는 점화식을 화이트보드에 먼저 쓰고 코드로 옮기기. 완전탐색은 "왜 브루트포스로도 시간 안에 되는가"(제약조건 n 크기) 근거부터 확인

## 주 단위 페이스 (study-plan.md와 동일 기준)
- 주 3~4문제, 하루 30~40분
- 순서: 우선순위1(15문제) 먼저 전부 완료 → 우선순위2(11문제) → 우선순위3(6문제, 시간 남으면)

- **일반 "경력직 채용절차" 안내 페이지(cjolivenetworks.co.kr/recruit/procedure)엔 코딩테스트가 명시 안 됨** — 이건 참고만, 실제 이 공고 원문이 우선.
- **"코딩테스트 없앴다"는 뉴스는 다른 회사 얘기**: 검색 중 나온 "올리브영, IT 개발자 채용…코딩테스트·사전 과제 없다" 기사는 **CJ올리브영**(CJ ONE과 별개 법인) 경력직 얘기임 — 지원 대상인 **CJ올리브네트웍스**와는 다른 회사라 착각하면 안 됨.
- **경력직 전용 코딩테스트 후기, 정정**: 웹 검색으로는 못 찾았으나(2026.07.22 시점), 이후 본인이 제공한 **잡플래닛 실제 후기 스크린샷(개발|과장, 2022-06, 합격)**에서 확인됨 — 상세는 `interview-prep-notes.md` "⭐ 최우선 자료 — 잡플래닛 실제 후기 스크린샷 10건" #8 참고.

### ⚠️ 중요 정정 — 경력직 코딩테스트는 "알고리즘이 아닌 실무 코딩"일 수 있음 (2026.07.22, 잡플래닛 실제 후기 기준)

> 개발|과장, 2022-06 합격 후기 원문: "경력직 코딩 테스트는 알고리즘이 아닌 실무적 업무 코딩으로 진행되었으며, 인터뷰는 진행 업무에 대한 깊이있는 질문과 대답, 트러블슈팅 내용까지 진행되어 좋은 인상이었음."

- 지금까지 이 파일에서 준비해온 "프로그래머스 알고리즘 문제(해시/BFS/그리디 등)" 전제와 **다를 수 있음** — 아래 유형·난이도 정보는 신입 공채 후기 기반이라 경력직에는 안 맞을 가능성이 있다는 뜻.
- **회차·직무마다 다를 수 있어 확정은 아님** — 알고리즘형/실무형이 섞여 있을 수 있으니, 아래 우선순위1~3 알고리즘 문제 감은 유지하되(비용이 적게 드는 준비라 밑져야 본전), 추가로 다음도 같이 준비해둘 것:
  - **실무형 코딩 대비**: "주어진 요구사항을 API 하나로 구현하기"(CRUD, 간단한 검증 로직, 문자열/데이터 가공을 실무 코드 스타일로), Java/Spring 기준으로 짧은 시간 안에 깔끔한 메서드/클래스 설계하는 연습
  - **코딩 후 인터뷰(트러블슈팅) 대비**: 코딩테스트 자체보다 그 코드를 두고 "왜 이렇게 짰는지, 트러블슈팅 경험은?"을 깊게 묻는 방식일 수 있음 — `study-plan.md`의 "설명 연습 템플릿"(문제 재진술→접근 후보 비교→선택 이유→복잡도)을 실무 코드에도 그대로 적용해 연습할 것
  - **면접 자체의 "장애 대응/트러블슈팅" 질문**과 강하게 연결되는 정보임 — `interview-prep-notes.md`의 "Q. 가장 기억에 남는 장애는?" 답변안과 세트로 준비.
- **결론**: 알고리즘 문제풀이(우선순위1~3)를 손 놓지 말고, 시간이 되면 실무형 코딩(Spring 기반 간단한 API 구현)도 병행 준비하는 게 안전.

### 전형·문제 수 추가 확정 (2026.07.22, 잡플래닛 후기 #21)

> 개발|대리, 2022-04, 헤드헌터 경로, 합격 후기 원문: "경력직 면접으로 코테 > 역량면접 > 임원면접 봤습니다. 코테는 2문제, 역량/임원 각각 3,40분 정도 봤습니다."

- **경력직 코테는 "2문제"라는 구체 수치가 나온 첫 사례** — 아래 "2시간 2~3문제(직무별 상이)"라는 기존 정보와 큰 충돌은 없지만, 신입 후기 기반이던 "2시간 3문제"보다는 경력직은 2문제 쪽에 무게를 두는 게 안전. 문제 수가 적으면 문제당 난이도나 실무형 비중이 상대적으로 커질 수 있다는 뜻이라, 시간 배분 연습할 때 "2문제 기준"으로도 한번 시뮬레이션해볼 것.
- **1차/2차라는 표현 대신 "역량면접/임원면접"으로 부르는 경우도 있음** — 같은 단계를 가리키는 다른 이름일 뿐이니 헷갈리지 말 것(`interview-prep-notes.md`도 동일하게 반영).

- **형식**: 프로그래머스 플랫폼, 2시간에 2~3문제(직무별 상이) — Software Engineer 계열 후기는 2시간 3문제, 경력직은 2문제 사례 확인됨(위 참고)
- **난이도**: 카카오·네이버보다 쉬운 편. 앞 1~2문제는 프로그래머스 레벨1~2 / 백준 실버1~2급, 마지막 문제가 골드4~5급 정도로 살짝 올라감 — 이 저장소 기준 "1. 문자열/배열"·"2. 스택/큐" 유형 위주로 감 유지하고, 마지막 한 문제 대비용으로 "3. BFS/DFS"까지만 추가로 다지면 충분해 보임(DP·그리디는 우선순위 낮춰도 됨)
- **언어**: Java 지원 확인됨(당일 선택 가능 언어가 4~5개로 제한된다는 후기 있음 — Java는 대부분 포함되는 주요 언어라 문제 없을 것)
- **함께 진행**: 코딩테스트와 별도로 프로그래머스 플랫폼에서 인성검사도 진행 — 코딩테스트 당일 일정에 포함될 수 있음, 시간 여유 두고 응시할 것

## 문제 풀 때마다 하는 "설명 연습" 템플릿

문제를 풀고 나서, 아래 4단계를 소리 내어 말하는 것까지가 한 세트. 코드만 통과시키고 넘어가면 코딩테스트 감은 늘어도 면접 설명력은 안 늘어남.

1. **문제 재진술**: "이 문제는 결국 OO를 OO 조건 안에서 찾는 문제입니다" — 면접관이 문제를 정확히 이해했는지 확인하는 단계이기도 함
2. **접근 후보 비교**: "완전탐색으로 하면 O(n²)인데, 제약조건상 n이 커서 시간초과 날 수 있어 O(n log n)으로 줄일 방법을 생각했습니다" — 브루트포스부터 언급하고 최적화 근거를 설명하는 순서가 면접에서 좋은 인상을 줌
3. **선택한 자료구조/알고리즘과 이유**: "그래서 해시맵을 써서 조회를 O(1)로 줄였습니다" 식으로 왜 그 도구를 골랐는지 한 문장
4. **시간/공간복잡도 정리**: 최종 풀이의 Big-O를 말하고, 트레이드오프(공간을 더 써서 시간을 줄였는지 등)가 있으면 언급
