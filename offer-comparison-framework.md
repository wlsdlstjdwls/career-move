# 처우 협상 — 시장가 리서치 & 오퍼 비교 기준

`job-change-checklist.md` 섹션 7 대응. 실제 개인 연봉 숫자는 이 파일에 적지 않음 — 그건 `comp-notes-private.md`(git에 커밋 안 되는 비공개 파일)에 따로 관리. 이 파일은 공개돼도 무방한 일반 정보만 정리.

## 시장가 리서치 (2026.07.20 조사, 참고용)

> **주의**: 아래는 공개 블로그/커뮤니티 통계를 종합한 것으로, 출처별 편차가 크고 신뢰도가 높지 않음(예: "6~7년차 5,560만~8,357만" 같은 범위와 "1억+" 사례가 동시에 나오는 등 표본이 뒤섞여 있음). **실제 협상 기준으로 쓰지 말고**, 아래 방법으로 본인 스펙에 맞는 데이터를 직접 확인할 것:
> 1. 원티드 인사이트(로그인 필요) — 연차·직무·기술스택 기준 연봉 분포 확인 가능
> 2. 크레딧잡 — 특정 회사 재직자 평균 연봉 확인 가능(타겟 회사별로 검색)
> 3. 잡플래닛 연봉 정보 — 회사별 리뷰와 함께 연봉 데이터 확인 가능
> 4. 블라인드 — 직군/연차별 익명 연봉 공유 글, 최신 글 위주로 참고(오래된 글은 시세 반영 안 됨)

일반적으로 통용되는 방향성 정도만 참고:
- 경력 6~7년차 백엔드는 시니어로 분류되는 구간의 시작점 — 대기업 계열사/중견기업 기준으로는 연차 대비 나쁘지 않은 처우를 받을 수 있는 구간이라는 게 여러 소스의 공통 의견
- 같은 연차라도 도메인(이커머스·핀테크·게임 등)과 회사 규모에 따른 편차가 매우 크다는 점이 공통적으로 언급됨 — "6년차 평균 얼마"보다 "이 회사, 이 직무 기준으로 얼마"가 훨씬 의미 있는 질문

## 희망 연봉 레인지 잡는 법

1. 현재 총 처우(기본급+성과급 등 실수령 기준)를 정확히 계산 (→ `comp-notes-private.md`)
2. "최소 이 정도는 올라야 이직 의미가 있다"는 하한선 설정 — 처우가 이직 사유 중 하나였으므로(◦`interview-prep-notes.md` "재입사 후 왜 또 이직" 답변 참고) 최소 상승폭 기준을 스스로 명확히 할 것
3. 원티드 인사이트/크레딧잡에서 확인한 개인 맞춤 데이터로 목표선·상한선 조정
4. 면접에서 "희망 연봉이 어떻게 되세요?"를 물으면 레인지로 답하되(예: "OOOO~OOOO 정도를 생각하고 있습니다"), 하한선보다 살짝 위를 레인지 하단으로 제시하는 게 일반적인 협상 전략

## 복수 오퍼 비교 기준 (오퍼가 여러 개 나올 경우)

연봉만으로 비교하면 놓치는 게 많음 — 아래 축을 다 놓고 비교할 것:

| 기준 | 확인할 질문 |
|------|-------------|
| 총 보상 | 기본급 + 성과급/인센티브(고정 vs 변동) + 스톡옵션/RSU(행사 조건까지) |
| 기술 스택 성장성 | 이 회사에서 3~5년 후 이력서에 뭐가 남는가 — 레거시 유지보수뿐인지, 최신 스택/아키텍처 경험이 쌓이는지 |
| 조직 안정성 | 계열사 편입/매각 이슈, 최근 구조조정 이력(블라인드/뉴스 확인) |
| 워라밸 | 야근 빈도, 온콜 부담 — 커피챗/후기로 확인(`networking-templates.md` 참고) |
| 역할 범위 | 설계 주도권을 실제로 가질 수 있는 포지션인지, 아니면 지시받은 기능만 구현하는 자리인지 |
| 성장 궤적 | 이 회사 경력이 다음 이직 때 어떤 근거가 되는지(예: "네이버/쿠팡급 경험"이 갖는 시장 가치) |

**진행 방법**: 오퍼가 나올 때마다 `comp-notes-private.md`의 "회사별 오퍼 비교" 표에 채워서 나란히 비교. 연봉 숫자가 가장 눈에 띄어도, 위 표의 다른 항목에서 압도적으로 밀리는 곳이면 최종 판단은 신중하게.
