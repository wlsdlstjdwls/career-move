# 기술 인터뷰 학습 계획 — 문제풀이 + 개념 정리

6년차 백엔드(Java/Spring/MyBatis) 기준. "이미 실무에서 써본 것"을 인터뷰에서 설명 가능한 수준으로 다듬는 데 집중 — 모르는 새 이론을 처음부터 배우는 게 아니라, 아는 걸 정확한 용어로 말하는 연습이 핵심.

## 1. 코딩테스트 (문제풀이)

- **목표 레벨**: 프로그래머스 Lv2~3, 백준 실버 상~골드 중
- **페이스**: 주 3~4문제, 하루 30~40분이면 충분 (한 번에 몰아서 안 함)
- **우선 유형**: 문자열/배열, 스택·큐, BFS/DFS, DP 기초, 그리디 — 이 5개만 잘해도 대부분의 코테 통과
- **주의**: 6년차는 "다 풀었나"보다 "설명을 깔끔하게 하나"가 더 중요 — 풀고 나서 시간복잡도/왜 이 자료구조를 골랐는지 말로 설명하는 연습까지 세트로
- **실전 문제 리스트 + 설명 템플릿 → `coding-test-practice.md`** (유형별 대표 문제, 주 단위 페이스, 진행상황 표 포함)

## 2. CS 개념 정리 (용어를 정확히 말할 수 있게)

체크박스는 "설명 가능 여부" 기준. 이미 실무 경험 있는 항목은 빠르게 훑고, 애매한 것 위주로 시간 투자.

> **답변 초안 전체 작성 완료 → `interview-prep-notes.md`** ("CS 기본기 — 설명 답변 초안" 섹션). 아래 체크박스는 "본인 언어로 소리 내어 설명 연습까지 끝냈는지" 기준으로 직접 체크할 것 — 초안이 있다고 자동으로 체크된 건 아님.

### 네트워크
- [ ] TCP 3-way handshake, TCP vs UDP
- [ ] HTTP/HTTPS, TLS handshake 개요
- [ ] REST 원칙, 상태코드 4xx/5xx 구분 기준

### OS
- [ ] 프로세스 vs 스레드, 컨텍스트 스위칭
- [ ] 동기/비동기, 블로킹/논블로킹 차이
- [ ] 데드락 발생 조건 4가지

### DB
- [ ] 인덱스 동작 원리 (B+Tree), 인덱스가 오히려 느려지는 경우
- [ ] 트랜잭션 격리수준 4단계 + 각 단계에서 발생 가능한 문제
- [ ] 락 종류 (Shared/Exclusive, 낙관적/비관적) — Fitness In Service 결제/세션 도메인 경험과 엮어서 설명 준비

### 자료구조/알고리즘
- [ ] 시간복잡도 Big-O 기본 케이스
- [ ] HashMap 내부 동작 (해시충돌 처리)

## 3. Java/Spring 심화 (실무 경험 기반 정리)

이미 써본 것들이라 "왜 그렇게 선택했는지"를 한 문장씩 정리해두는 게 목표.

> **답변 뼈대 초안 완료 → `interview-prep-notes.md`** ("Spring/Java 심화 — 답변 초안"). 단, `(확인 필요)` 표시 부분은 실제 경험으로 본인이 직접 채워야 완성됨 — 이 항목 하나하나는 채워 넣은 뒤 체크.

- [ ] MyBatis vs JPA — Fitness In Service/커머스웨어에서 MyBatis 쓴 이유, JPA였다면 뭐가 달라졌을지
- [ ] 트랜잭션 전파(Propagation) — REQUIRES_NEW를 써야 했던 실제 케이스 있었는지 점검
- [ ] Spring Batch — Job/Step/Tasklet 구조, Fitness In Service의 SessionUpdateTasklet 사례로 설명
- [ ] JWT — Access/Refresh 토큰 전략, 왜 BO는 1시간/MO는 7일로 다르게 뒀는지
- [ ] Redis — 캐싱 전략, Spring Session에 왜 Redis를 썼는지
- [ ] 멀티모듈 구조 — core/bo-api/mo-api/batch로 나눈 이유, 얻은 이점

## 5. Kubernetes 기초 (CJ ONE 지원 갭 대응)

CJ ONE 백엔드 공고 필수요건 중 유일한 갭. 실무 경험을 만들 수는 없으니, 면접 꼬리질문에 막히지 않을 정도의 개념만 확실히 잡는 게 목표 — `interview-prep-notes.md`의 K8s 답변안에서 예고된 "Pod가 뭔지 설명해보실 수 있나요?" 같은 질문에 대비.

- [ ] Pod/Service/Deployment 3개 리소스 — 각각 뭘 하는지, Docker 컨테이너와 Pod의 차이 (Pod가 컨테이너를 감싸는 한 단계 위 추상화)
- [ ] 롤링업데이트 — 무중단 배포와 어떻게 연결되는지 (Docker+Nginx로 해봤던 무중단 전환 경험과 비교해서 설명할 수 있게)
- [ ] 오토스케일링(HPA) 개념 — 트리거 조건, Pod가 늘어나는 흐름
- **페이스**: 실무 경험 없이 개념만 익히는 거라 하루 20~30분씩 3~4일이면 충분 — 무료 인프런/유튜브 "쿠버네티스 입문" 급 강의 하나로 개념만 훑는 정도로 족함, 자격증 수준 필요 없음
- **목표 시점**: 서류 제출(07.27~28 목표) 전까지 최소 1회독 — 서류 통과 후 1차 면접 전에 한 번 더 복습
