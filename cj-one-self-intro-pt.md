```
[이름] — 백엔드 개발자 (Java/Spring, 6년차)

■ E-커머스 "고객 자산" 도메인 전문
   상품·주문·프로모션·클레임 + 예치금/포인트/바우처 API 설계·운영

■ 대규모 트래픽 환경 안정성 경험
   더현대·투홈 플랫폼 통합(NCP), 무중단 정책 대응

■ 개인 프로젝트로 증명한 End-to-End 역량
   Fitness In Service — 기획→개발→배포까지 단독 수행
```

speaker note (약 30~35초):
> 안녕하십니까, 백엔드 개발자 [이름]입니다. 6년간 이커머스 도메인에서 상품·주문·프로모션·클레임을 다뤄왔고, 최근에는 더현대·투홈 플랫폼 통합 프로젝트에서 예치금·포인트·바우처 같은 "고객 자산" API를 직접 담당했습니다. CJ ONE이 다루는 통합 멤버십·포인트 도메인과 맞닿아 있는 경험이라 이 자리에 지원하게 됐습니다.

## 슬라이드 2 — 대표 경험 2가지

```
1) 대규모 트랜잭션 안정성
   더현대·투홈 통합 — 바우처 정책 변경으로 옵션 포맷 가변화
   → 영향 쿼리 전수조사 + 정규식 일괄 교체, 무중단 반영

2) End-to-End 역량 (Fitness In Service)
   Spring Boot 멀티모듈(BO/MO/Batch) 단독 설계
   JWT·Redis 세션·Kakao OAuth·Toss 결제·SSE 알림
   AWS EC2/Docker/Nginx 배포, 실서비스 운영 중
```

speaker note (약 40~45초):
> 더현대·투홈 통합 당시 오픈 직전 바우처 정책이 변경되면서 옵션 포맷이 가변화돼 파싱 오류가 발생했는데, 영향받는 쿼리를 전수 조사하고 정규식 기반으로 일괄 교체한 뒤 서비스 중단 없이 반영했습니다. 이런 실무 경험과 별개로, 개인 프로젝트 Fitness In Service에서는 회원 앱을 Spring Boot 멀티모듈 구조로 설계해 JWT 인증, Redis 세션, 결제, 실시간 알림까지 직접 구현하고 AWS에 배포해 운영하고 있습니다. 기획부터 배포까지 책임진 경험으로 사용자 관점의 설계 감각을 키웠습니다.
